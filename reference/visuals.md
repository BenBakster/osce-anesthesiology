# Visual components — бриф

Опис трьох ключових візуальних блоків застосунку. Claude Code має згенерувати їх як inline SVG (для двох схем) та HTML-таблицю (для антидотів), використовуючи дослівні тексти з `content/scenarios.md`.

---

## Загальні вимоги до всіх трьох

- Inline SVG / HTML, без зовнішніх бібліотек
- Адаптив: на мобільному (`≤600px`) — вертикальний стек
- Light / dark тема через CSS variables — детект через `prefers-color-scheme`
- Семантика кольорів — стабільна між компонентами
- Шрифти системні sans-serif (`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`)
- Без емодзі-іконок та декоративних елементів
- Доступність: `role="img"` + `<title>` + `<desc>` для SVG, `aria-label` для інтерактивних вузлів

## Палітра (семантична)

| Призначення | Світла тема | Темна тема |
|---|---|---|
| ФШ / ШТ, дефібриляційний ритм | coral / світло-червоний | приглушений coral |
| Асистолія / БЕА, недефібриляційний ритм | синій | приглушений синій |
| Оборотні причини, увага | янтарний | приглушений янтарний |
| ROSC, успіх | бірюзовий / зелений | приглушений бірюзовий |
| Нейтральні / структурні вузли | сірий | темно-сірий |

Реалізувати через CSS variables (`--color-shockable-bg`, `--color-shockable-stroke`, `--color-shockable-text` тощо). Контрастність — мінімум WCAG AA (4.5:1 для тексту).

## Схема 1: основний алгоритм ALS

### Призначення
Карта дій від моменту констатації зупинки серця до ROSC. Демонструє розгалуження за типом ритму та повернення до пошуку оборотних причин.

### Розташування на сторінці
Головна сторінка (`#general` або корінь). Перший блок одразу після заголовка.

### Структура
Вертикальна. Спільний верхній блок → точка розгалуження → дві паралельні колонки → сходження → спільний нижній блок.

### Вузли (зверху вниз)

#### Спільний верхній блок (нейтральний / сірий)
1. **Зупинка серця**
2. **СЛР 30:2 + ЗІЗ + дефібрилятор**
3. **Оцінка ритму** ← точка розгалуження

#### Ліва колонка (coral) — дефібриляційний ритм
4. **ФШ / ШТ без пульсу**
5. **Розряд 150 Дж**
6. **СЛР 2 хв**
7. **Після 3-го розряду** / *аміодарон 300 + адреналін 1 мг*
8. **Після 5-го розряду** / *аміодарон 150 мг повторно*

#### Права колонка (blue) — недефібриляційний ритм
4. **Асистолія / БЕА**
5. **Адреналін 1 мг негайно**
6. **СЛР 2 хв**
7. **Повторно адреналін** / *1 мг кожні 3–5 хв*
8. **Переоцінка ритму** / *кожні 2 хв*

Вузли 7 і 8 — двострічкові: жирний заголовок + приглушений підпис нижче.

#### Спільний нижній блок
9. **4Г 4Т: паралельно шукаємо оборотну причину** (amber, ширший за решту)
10. **ROSC → післяреанімаційна допомога** (teal)

### Зв'язки
- Вертикальні стрілки між сусідніми вузлами всередині кожної колонки
- Дві діагональні стрілки від "Оцінка ритму" до перших вузлів обох колонок
- Дві діагональні стрілки від останніх вузлів обох колонок до "4Г 4Т"
- Вертикальна стрілка від "4Г 4Т" до "ROSC"
- Стрілки — тонкі (`stroke-width: 1`), із наконечником-шевроном

### Інтерактивність
- Клік на "4Г 4Т" → перехід на маршрут `#4g4t`
- Клік на "ФШ / ШТ" → фільтр списку сценаріїв за дефібриляційним ритмом
- Клік на "Асистолія / БЕА" → фільтр за недефібриляційним ритмом
- Решта вузлів — статичні (без курсору-вказівника)
- Hover-стан тільки на клікабельних: легке затемнення фону

### Розміри і пропорції
- viewBox: `0 0 680 720` (десктопна версія)
- Вузли в спільному блоці: ширина 320px, висота 44px, центровані
- Вузли в колонках: ширина 260px, висота 44px (одностічкові) / 56px (двостічкові)
- Колонки розміщені симетрично відносно центру (x=40–300 ліва, x=380–640 права)
- "4Г 4Т" — ширина 480px, "ROSC" — ширина 380px
- Радіус заокруглення кутів: 8px

### Адаптив
≤600px — вертикальний стек:
1. Спільний верхній блок (3 вузли)
2. Розгалуження як горизонтальна смужка-перемикач: "ФШ/ШТ" ↔ "Асистолія/БЕА"
3. Активна колонка показується повністю (5 вузлів), друга прихована
4. Спільний нижній блок (2 вузли)

Альтернатива: показувати обидві колонки одна під одною з чіткими заголовками.

## Схема 2: карта 4Г 4Т

### Призначення
Швидкий довідник: оборотна причина → сценарій → ключове втручання. Дозволяє за один погляд зорієнтуватися перед усною відповіддю.

### Розташування на сторінці
Окремий маршрут `#4g4t`. Також може бути вбудована як швидкий перегляд на головній.

### Структура
Сітка 2 × 4: дві горизонтальні смуги по чотири картки. Заголовок секції над кожною смугою.

### Картки

#### Верхня смуга — 4Г метаболічно-фізіологічні (blue)

Заголовок секції: **«4Г — метаболічно-фізіологічні»**

| Назва | Сценарій | Лікування |
|---|---|---|
| Гіпоксія | Сц. 1 | 100% O₂ + адреналін 1 мг |
| Гіповолемія | Сц. 6 | Кров 1:1:1, транексам 1 г |
| Гіпокаліємія | Сц. 4 | KCl 2 ммоль/хв + MgSO₄ 2 г |
| Гіпотермія | Сц. 3 | Зігрівання, без адреналіну |

#### Нижня смуга — 4Т тромбо-механічно-токсичні (coral)

Заголовок секції: **«4Т — тромбо-механічно-токсичні»**

| Назва | Сценарій | Лікування |
|---|---|---|
| Тромбози | Сц. 2, 5, 7 | ПКВ або тромболізис |
| Тампонада | Сц. 8 | Перикардіоцентез під УЗД |
| Токсини | Сц. 9.1, 9.2 | Антидот за агентом |
| Пневмоторакс | Сц. 10 | Голкова декомпресія |

### Структура картки
Три текстові ряди:
1. Назва причини — заголовок, шрифт 14px medium, центрований
2. Номер сценарію — підзаголовок, шрифт 12px, приглушений колір
3. Лікування — два рядки тексту по 12px, приглушений колір (треба для довгих фраз типу "Перикардіоцентез під УЗД")

### Розміри
- viewBox: `0 0 680 360` (десктоп)
- Картка: 130 × 110px
- Інтервал між картками в ряду: 20px
- Інтервал між смугами: 60px (з місцем для заголовка)

### Інтерактивність
- Hover на картці: тонка рамка стає виразнішою, легке затемнення
- Клік:
  - Якщо один сценарій — перехід на нього: `#scenario-1`, `#scenario-3` тощо
  - Якщо кілька (Тромбози → 2, 5, 7; Токсини → 9.1, 9.2) — модалка зі списком або перехід на головну з фільтром

### Адаптив
- ≤900px: 2 × 4 → 4 × 2 (дві картки в ряду)
- ≤600px: 4 × 2 → 8 × 1 (по одній картці в ряду, на повну ширину)

При зменшенні до однієї колонки картки розширюються по горизонталі, заголовок і вміст вирівнюються по лівому краю замість центру.

## Таблиця 3: антидоти

### Призначення
Довідник антидотів за класами токсинів. Фокусована для усної відповіді про сценарії 9.1 та 9.2 і додаткові питання викладача.

### Розташування на сторінці
Окремий маршрут `#antidotes`. Також може бути вбудована як секція внизу маршруту сценарію 9.2.

### Структура
HTML-таблиця з групувальними рядками. Три колонки: **Токсин** | **Антидот** | **Доза / примітка**.

### Зміст

#### Група 1: Кардіотоксичні

| Токсин | Антидот | Доза / примітка |
|---|---|---|
| ТЦА | Натрію гідрокарбонат 8,4% | 1–2 ммоль/кг в/в болюсом, цільовий pH 7,50–7,55 |
| Дигоксин | Fab-фрагменти антитіл (DigiFab) | За нормограмою; не «антигени» |
| БКК (верапаміл, ділтіазем) | Кальцій + глюкагон + інсулін | Ca глюконат 10% — 10 мл; глюкагон 5–10 мг; HIET |
| β-блокатори | Глюкагон + кальцій | Глюкагон 5–10 мг в/в, далі інфузія |
| Місцеві анестетики (LAST) | Ліпідна емульсія 20% | 1,5 мл/кг болюс, далі 0,25 мл/кг/хв |

#### Група 2: Депресанти ЦНС

| Токсин | Антидот | Доза / примітка |
|---|---|---|
| Опіати | Налоксон | 0,4–2 мг в/в, повтор 2–3 хв, до 10 мг |
| Бензодіазепіни | Флумазеніл | 0,2 мг в/в (обережно — судоми при мікст-отруєнні) |

#### Група 3: Холінергічні

| Токсин | Антидот | Доза / примітка |
|---|---|---|
| ФОС | Атропін + пралідоксим | Атропін 2 мг кожні 5 хв до атропінізації |

#### Група 4: Метаболічні

| Токсин | Антидот | Доза / примітка |
|---|---|---|
| Метанол | Етанол або фомепізол | Етанол 1,5 мл/кг 40% per os + фолат + NaHCO₃ |
| Парацетамол | N-ацетилцистеїн | 150 мг/кг за 1 год → 50 мг/кг за 4 год → 100 мг/кг за 16 год |
| Ізоніазид | Піридоксин (вітамін B6) | 1 г піридоксину на 1 г ізоніазиду |

### Стилізація
- Заголовки колонок: 12px, приглушений колір, медіум-вага, нижня рамка
- Назва токсину: 14px, regular, темний колір (основний колір тексту)
- Антидот: 14px, regular, темний колір
- Доза / примітка: 13px, regular, приглушений колір
- Групувальні рядки: 12px, приглушений колір, медіум-вага, без рамки, відступ зверху 16px
- Розділювальні лінії: тонкі (`1px solid var(--color-border)`), без альтернативного фону рядків (без зебри)
- Внутрішні відступи комірок: 10px вертикалі × 12px горизонталі

### Інтерактивність
Без інтеракції. Чисто статичний довідник. Винятки:
- На мобільному рядки можуть розкриватися (accordion) — клік на токсин → розгортається антидот і доза. Опціонально, тільки якщо стандартна таблиця погано читається на вузькому екрані

### Адаптив
- ≤768px: зменшити шрифти на 1px, скоротити горизонтальні відступи
- ≤480px: можна перейти на двоколонковий вигляд (об'єднати "Антидот" і "Доза" в одну комірку з переносом рядка) або акордеон як зазначено вище

## Стилістичний орієнтир

[`benbakster.github.io/ampd-tools`](https://benbakster.github.io/ampd-tools/) — стримана медична типографіка, без декоративних елементів, акуратні відступи, чіткі межі секцій.

Цільовий стиль:
- Білий / м'який кремовий фон у світлій темі
- Темно-сірий, не чорний, основний текст
- Тонкі лінії та рамки замість заливок
- Мінімум тіней (краще без них зовсім)
- Заокруглення помірне (4–8px), не пілюлеподібне

## Розміщення в коді

```
src/
├── components/
│   ├── ALSAlgorithm.html      ← Схема 1 (inline SVG в HTML-обгортці)
│   ├── ReversibleCauses.html  ← Схема 2
│   └── AntidotesTable.html    ← Таблиця 3
├── styles/
│   └── visuals.css            ← Спільні стилі для трьох візуалок
└── data/
    └── scenarios.js           ← Структуровані дані сценаріїв
```

Якщо застосунок — SPA на vanilla JS, компоненти можуть бути функціями, що повертають DOM-вузли або HTML-рядки. Якщо буде вибрано компонентний підхід (Lit, Web Components) — зробити три окремі custom elements.
